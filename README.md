## 1. Тема и концепция

**Сеть моих коттеджей «Гордеют»**

* Главная цель — разработать веб-приложение для просмотра каталога коттеджей и выбора наиболее подходящего варианта для отдыха.
* Бронирование — сервис позволяет забронировать домик на выбранные даты с автоматическим контролем соответствия условиям аренды.
* Пользователи могут просматривать подробную информацию о коттеджах, включая фотографии, описание, стоимость проживания и доступные удобства.
* Система предоставляет удобный интерфейс для поиска и фильтрации объектов по различным параметрам.

**Целевая аудитория:**
* незарегистрированные пользователи, просматривающие каталог;
* зарегистрированные пользователи, оформляющие бронирование;
* администратор, управляющий объектами и заявками.

**Ключевой функционал:**
* просмотр каталога коттеджей;
* поиск и фильтрация по параметрам;
* просмотр детальной информации о коттедже;
* бронирование на выбранные даты;
* управление бронированиями через личный кабинет;
* регистрация и авторизация пользователей.

## 3. Технологический стек

**Frontend**

* Фреймворк: React + Vite — современная и производительная связка для разработки одностраничных веб-приложений.
* Маршрутизация: React Router — организация навигации между страницами с поддержкой публичных и защищённых маршрутов.
* State-менеджмент: React Context API (AuthProvider, CottageProvider) — централизованное управление состоянием авторизации пользователей и данными коттеджей.
* Уведомления: react-hot-toast — отображение всплывающих уведомлений об успешных действиях и ошибках.

**Backend**

* Среда выполнения: Node.js — платформа для обработки клиентских запросов и выполнения серверной логики.
* Фреймворк: Express.js — создание REST API для взаимодействия клиента с сервером.
* База данных: MongoDB — документо-ориентированная СУБД для хранения информации о пользователях, коттеджах и бронированиях.

**Инфраструктура**

* Хостинг: Railway — облачная платформа для развертывания Frontend и Backend с автоматическим деплоем из репозитория.
* Контроль версий: Git / GitHub — система управления исходным кодом и организации процесса разработки.

## 4. Визуальное проектирование интерфейса

## 4. Визуальное проектирование интерфейса

**Главная (/)**
* Тип доступа: публичный
* Основные блоки:
    1. **Боковая панель (Sidebar)** — навигационное меню («Обзор», «Поиск») и раздел кабинета («Настройки», «Войти»).
    2. **Поисковая панель** — блок «Куда поедем?» для быстрого поиска и фильтрации доступных объектов.
    3. **Секция каталога** — список карточек коттеджей с основной информацией (название, описание, стоимость проживания).
    4. **Пагинация** — элементы управления для перехода между страницами каталога.
* Динамический контент:
    * Список объектов загружается с сервера через API.
    * Для управления данными используется `CottageContext`, обеспечивающий централизованное хранение и обновление информации о коттеджах.
    * При изменении параметров поиска или переключении страницы каталог обновляется автоматически.

**Аутентификация (/login, /register)**
* Тип доступа: публичный
* Основные блоки:
    1. **Вход в систему** — поля для ввода email и пароля, чекбокс «Запомнить меня», ссылка для восстановления доступа.
    2. **Регистрация** — форма создания аккаунта с полями: имя пользователя, email, пароль, ключевое слово для восстановления и необязательная подсказка.
    3. **Восстановление доступа** — форма сброса пароля с вводом email, ключевого слова и нового пароля.
* Динамический контент:
    * Интерактивная проверка корректности вводимого пароля с отображением подсказок и сообщений об ошибках.
    * Автоматическая подгрузка подсказки для ключевого слова на основе введённого email с использованием `useEffect` и механизма `debounce`.
    * Отображение уведомлений об успешных операциях и ошибках через библиотеку `react-hot-toast`.
    * Обработка и отображение ошибок валидации, полученных от сервера.

**Поиск (/search)**
* Тип доступа: публичный
* Основные блоки:
    1. **Панель расширенного поиска** — форма выбора области, города или населённого пункта, дат заезда и выезда, а также количества гостей.
    2. **Интерактивная карта** — отображение доступных коттеджей на карте для удобного поиска объектов по местоположению.
    3. **Результаты поиска** — список найденных коттеджей в виде карточек с фотографиями, кратким описанием и стоимостью проживания.
    4. **Счётчик результатов** — отображение общего количества объектов, соответствующих заданным параметрам поиска.
* Динамический контент:
    * Результаты поиска обновляются в режиме реального времени на основе выбранных пользователем фильтров.
    * Данные о доступных объектах загружаются с сервера через API.
    * Интеграция с Яндекс.Картами обеспечивает отображение местоположения коттеджей на карте.
    * При изменении параметров поиска автоматически выполняется повторный запрос и обновление списка объектов.

**Детальная страница (/cottage/:id)**
* Тип доступа: публичный
* Основные блоки:
    1. **Основная информация об объекте** — крупное изображение коттеджа, название и подробное описание.
    2. **Характеристики** — отображение ключевых параметров объекта, включая вместимость, местоположение и дополнительные удобства.
    3. **Панель бронирования** — боковая форма для выбора дат заезда и выезда с использованием календарного виджета.
    4. **Кнопка бронирования** — подтверждение выбора дат и переход к оформлению брони.
* Динамический контент:
    * Данные о коттедже загружаются с сервера по уникальному идентификатору объекта (`id`).
    * Информация об объекте обновляется на основе данных, полученных через API.
    * Интерактивный календарь позволяет выбрать доступные даты проживания.
    * Логика бронирования проверяет корректность выбранного периода и доступность объекта на указанные даты.

**Личный кабинет (/profile)**
* Тип доступа: приватный (требуется авторизация)
* Основные блоки:
    1. **Профиль пользователя** — отображение аватара с инициалами пользователя и заголовка страницы.
    2. **Информационная карточка** — вывод основной информации о пользователе, включая имя и адрес электронной почты.
    3. **Статистика активности** — отображение количества активных бронирований пользователя в виде информационного бейджа.
    4. **Управление аккаунтом** — кнопки для выхода из системы и удаления учётной записи.
* Динамический контент:
    * Данные профиля автоматически загружаются с сервера при открытии страницы.
    * Информация о пользователе отображается на основе текущей авторизованной сессии.
    * Количество активных бронирований обновляется на основании данных, полученных через API.
    * Состояние авторизации и актуальные данные пользователя управляются через `AuthContext`.
    * Уведомления об успешных действиях и ошибках отображаются с использованием `react-hot-toast`.

**Мои бронирования (/my-bookings)**
* Тип доступа: приватный (требуется авторизация)
* Основные блоки:
    1. **Список бронирований** — отображение карточек забронированных коттеджей с основной информацией об объекте и параметрах брони.
    2. **Информация о поездке** — данные о датах заезда и выезда, а также продолжительности проживания.
    3. **Статус бронирования** — отображение текущего состояния заявки (например: «На рассмотрении», «Подтверждено», «Отменено»).
    4. **Финансовая информация** — отображение полной стоимости аренды в белорусских рублях (BYN).
    5. **Управление бронированием** — возможность отмены активной заявки через кнопку «Отменить».
* Динамический контент:
    * Список бронирований пользователя загружается с сервера через API после авторизации.
    * Данные автоматически обновляются при создании, изменении или отмене бронирования.
    * Статусы заявок отображаются в соответствии с актуальной информацией, полученной от сервера.
    * После отмены бронирования интерфейс обновляется без перезагрузки страницы.
    * Уведомления об успешных действиях и ошибках отображаются с использованием `react-hot-toast`.

**Настройки (/settings)**
* Тип доступа: приватный (требуется авторизация)
* Основные блоки:
    1. **Редактирование профиля** — формы для изменения имени пользователя и адреса электронной почты.
    2. **Управление уведомлениями** — переключатель для настройки получения уведомлений и email-рассылок.
    3. **Сохранение изменений** — кнопка подтверждения и отправки обновлённых данных на сервер.
* Динамический контент:
    * Текущие данные пользователя автоматически загружаются с сервера при открытии страницы.
    * Поля формы заполняются актуальной информацией из профиля пользователя.
    * Перед отправкой выполняется проверка корректности введённых данных.
    * Изменённые настройки сохраняются через API без необходимости повторной авторизации.
    * После успешного обновления отображается уведомление с использованием `react-hot-toast`.
    * Состояние пользователя и его настройки синхронизируются через `AuthContext`.

**Добавление объекта (/add-cottage)**
* Тип доступа: приватный (требуется авторизация)
* Функциональность:
    * Создание нового объявления о сдаче коттеджа в аренду с использованием пошагового мастера настройки.
* Этапы создания объявления:
    1. **Основная информация** — ввод названия объекта, выбор типа недвижимости, указание количества комнат и максимального числа гостей.
    2. **Расположение** — выбор области, города или населённого пункта, а также ввод точного адреса объекта.
    3. **Описание и стоимость** — заполнение подробного описания коттеджа и указание стоимости аренды за сутки.
    4. **Загрузка медиафайлов** — выбор и загрузка главного изображения для объявления.
* Основные блоки:
    1. **Форма управления** — набор интерактивных полей ввода с проверкой корректности данных на каждом этапе заполнения.
    2. **Блок предпросмотра** — отображение будущей карточки объекта с автоматическим обновлением данных в режиме реального времени.
    3. **Навигация по шагам** — кнопки «Назад», «Далее» и «Сохранить» для управления процессом создания объявления.
* Динамический контент:
    * Состояние формы синхронизируется с блоком предпросмотра посредством механизма `React State`.
    * Изменения, внесённые пользователем, мгновенно отображаются в предварительном просмотре карточки объекта.
    * На каждом этапе выполняется проверка введённых данных перед переходом к следующему шагу.
    * После завершения заполнения формы данные объекта отправляются на сервер через API и сохраняются в базе данных.
    * Результат операции сопровождается уведомлениями об успешном создании объекта или возникших ошибках с использованием `react-hot-toast`.

## 5. Схема данных и API

**1. Модели данных**

### User

| Поле | Тип | Описание |
|-------|------|----------|
| id | integer, PK | Уникальный идентификатор пользователя |
| name | string | Имя пользователя |
| email | string, unique | Email для входа в систему |
| password_hash | string | Хеш пароля |
| secret_word | string | Секретное слово для восстановления доступа |
| secret_word_hint | string | Подсказка к секретному слову |
| role | string | Роль пользователя (user / admin) |
| created_at | datetime | Дата регистрации |
| updated_at | datetime | Дата последнего обновления |

### Cottage

| Поле | Тип | Описание |
|-------|------|----------|
| id | integer, PK | Уникальный идентификатор коттеджа |
| name | string | Название коттеджа |
| rooms | integer | Количество комнат |
| type | string | Тип объекта (house / apartment) |
| min_guests | integer | Минимальное количество гостей |
| max_guests | integer | Максимальное количество гостей |
| region | string | Область расположения |
| city | string | Город |
| address | text | Точный адрес объекта |
| price_per_night | decimal | Стоимость проживания за ночь |
| description | text | Подробное описание объекта |
| media_urls | array/string | Ссылки на фотографии |
| is_popular | boolean | Популярный объект |
| created_at | datetime | Дата добавления |
| updated_at | datetime | Дата последнего обновления |

### Booking

| Поле | Тип | Описание |
|-------|------|----------|
| id | integer, PK | Уникальный идентификатор бронирования |
| user_id | FK → User.id | Пользователь, оформивший бронирование |
| cottage_id | FK → Cottage.id | Забронированный коттедж |
| start_date | date | Дата заезда |
| end_date | date | Дата выезда |
| total_price | decimal | Итоговая стоимость проживания |
| status | string | Статус бронирования (pending, confirmed, canceled) |
| created_at | datetime | Дата создания записи |

**2. Ключевые эндпоинты API**

### 1. Аутентификация и пользователи

| Метод | URL | Описание | Тело запроса | Ответ |
|--------|-----|----------|-------------|--------|
| POST | `/api/auth/register` | Регистрация пользователя | `{ name, email, password, secret_word, secret_word_hint }` | `{ id, name, email }` |
| POST | `/api/auth/login` | Вход в систему | `{ email, password }` | `{ id, name, email, token }` |
| POST | `/api/auth/recover` | Восстановление пароля | `{ email, secret_word, new_password }` | `{ message }` |
| GET | `/api/users/me` | Получение данных профиля | — | `{ id, name, email, role }` |

### 2. Каталог и коттеджи

| Метод | URL | Описание | Параметры / Тело | Ответ |
|--------|-----|----------|------------------|--------|
| GET | `/api/cottages` | Поиск и фильтрация коттеджей | `?region=&guests=&date_start=&date_end=` | `[ { id, title, price, location, images } ]` |
| GET | `/api/cottages/:id` | Получение информации о коттедже | — | `{ id, title, description, price, region, city, address, rooms, max_guests, media_urls }` |
| POST | `/api/cottages` | Создание нового объявления | `{ title, description, price, region, city, address, rooms, guests, media_urls }` | `{ id, status }` |

### 3. Бронирования

| Метод | URL | Описание | Тело запроса | Ответ |
|--------|-----|----------|-------------|--------|
| POST | `/api/bookings` | Создание бронирования | `{ cottage_id, start_date, end_date }` | `{ id, status, total_price }` |
| GET | `/api/bookings/me` | Получение списка бронирований пользователя | — | `[ { id, cottage_id, start_date, end_date, total_price, status } ]` |
| DELETE | `/api/bookings/:id` | Отмена бронирования | — | `{ message: "Booking canceled" }` |

## 7. Запуск проекта

Для локального запуска приложения необходимо выполнить следующие шаги.

### Предварительные требования
* **Node.js** — версия 18 или выше (рекомендуется LTS).
* **MongoDB** — установленная и запущенная база данных.
* **Git** — для клонирования репозитория.

### 1. Клонирование репозитория
```bash
git clone https://github.com/Marina-228/gordeut.git
cd gordeut
```

### 2. Настройка окружения
Создайте файл `.env` в корневой директории проекта и добавьте необходимые переменные окружения:
```env
PORT=5000
MONGODB_URI=mongodb://localhost:27017/gordeut
JWT_SECRET=your_super_secret_key
```
При необходимости используйте файл `.env.example` в качестве шаблона.

### 3. Установка зависимостей
Установите все необходимые зависимости проекта:
```bash
npm install
```

### 4. Подготовка базы данных
Убедитесь, что сервер MongoDB запущен и доступен по адресу, указанному в переменной `MONGODB_URI`.
При первом запуске коллекции базы данных будут созданы автоматически.

### 5. Запуск приложения
Запустите серверную часть:
```bash
npm run server
```
В отдельном терминале запустите клиентскую часть:
```bash
npm run dev
```

### 6. Доступ к приложению
После успешного запуска приложение будет доступно по следующим адресам:
| Сервис | URL |
|---------|---------|
| Frontend | `http://localhost:5173` |
| Backend API | `http://localhost:5000` |
После запуска пользователь сможет зарегистрироваться, просматривать каталог коттеджей, создавать объявления и оформлять бронирования.
